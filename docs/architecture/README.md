# Architecture Briefing Hub

Dieser Ordner bündelt die Architektur- und Produkt-Briefings als Arbeitsgrundlage für alle Teams.

## Primäre Quellen
- `docs/architecture/architecture.md` (MVP-Definition, Epics, Stories, Outcome)
- `docs/architecture/legaltech-produkt-briefing.md` (Zielbild, Rollen, Kern-Use-Cases)

## Arbeitsmodus
1. Architekturentscheidungen werden als ADR im zentralen Wissensbereich dokumentiert.
2. Teambriefings verweisen auf die Epics/Stories aus den Primärquellen.
3. Änderungen an Anforderungen werden zuerst hier als Delta notiert und danach in Team-Briefings verteilt.

## Delta-Log (Start)
- Fokus auf SME-Kanzleien mit <30-Minuten Vertragsdurchlauf.
- Strikte Mandantenfähigkeit + DSGVO + optional On-Prem als Leitplanken.
- Guided Contract Builder, Klausel-Konsistenz und Export als Kern-MVP.
