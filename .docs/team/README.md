# Team Communication Space

Unterordner in `.docs/team/` dienen als Kommunikationsräume für Dokumentation, Briefings und Team-Updates.

## Team-Aufstellung
1. `01-product-architecture`
2. `02-platform-security-identity`

4. `04-contract-builder`
5. `05-export-integration`
6. `06-qa-compliance`
7. `07-devops-onprem`

## Kommunikationsregeln
- Jedes Team pflegt mindestens:
  - `briefing.md` (Mandat, Scope, Deliverables)
  - `updates.md` (laufende Status-Updates)
- Entscheidungsrelevante Punkte werden zusätzlich im zentralen Wissensmanagement erfasst.
