# Briefing Snapshot – MVP Definition

Quelle: `docs/architecture/architecture.md`

## Ziel
Eine Kanzlei soll in unter 30 Minuten einen konsistenten Vertrag auswählen, ausfüllen, speichern und exportieren können.

## MVP-Kern (Epics)
1. Mandantenfähigkeit & Identity
2. Verlags-Content mit Versionierung/Publishing
3. Guided Contract Builder
4. Klausel-Konsistenz & Validierung
5. Dokumentmanagement + Export

## Leitplanken
- Tenant Isolation serverseitig erzwingen.
- Version Pinning für reproduzierbare Verträge.
- Publishing nur über definierte Freigabezustände.
